# 0601 改进讨论

## 当前 Loss 架构分析

### L_ans 是否包含 `</think>`？

**是的，包含。** 当前代码逻辑：

```python
# extract_think_content() 中：
answer_part = trajectory[think_end:]  # 包含 </think>
```

所以 `answer_ids` = `[</think>, <answer>, ..., </answer>, <|im_end|>]`

Phase 3 forward 时，这整段作为一个序列输入，用 shift 后的标准 next-token CE loss：
- 位置 0 (`</think>`) 的 logits 预测位置 1 的 token
- 位置 1 的 logits 预测位置 2 的 token
- ...

**问题**：`</think>` 的准确预测和后续 answer 内容的准确预测被混在同一个 loss 里，无法分别观察和调权。

### 建议改进：拆分 L_ans 为两部分

1. **L_think_end**：只关注 `</think>` token 的预测准确性（即最后一个 latent hidden 是否能正确输出 `</think>`）
2. **L_answer**：只关注 `</think>` 之后的 answer 内容预测准确性

这样可以在 wandb 上分别观察：
- 模型是否学会了"何时停止潜空间推理"
- 模型是否学会了"输出正确答案"

## 数据问题分析

用户反馈：训练数据可能存在 bug（感知教师视频问题 / 学生模型视频采样问题 / QA 对问题）。

从 loss 曲线看：
- `loss_trans` 下降很快 → 模型能学会潜空间编码（Translator 能还原）
- `loss_ans` 下降慢且最终值偏高 → 答案部分学不好

**推测**：数据中 GT answer 与视频内容不匹配（可能是视频路径错误、或感知教师生成的 result 与实际视频不对应），导致模型无法从视频特征中学到正确答案。

## 潜空间特征"走捷径"问题

### 当前架构的梯度流向

```
L_trans → Translator CE → 梯度穿过 h_i → Student 参数更新
L_ans   → Phase 3 CE → 梯度穿过 answer_embeds → Student embed_tokens 更新
                      → 但 KV cache 是 detach 的！梯度无法回传到 Phase 2 的 latent hiddens！
```

### 核心问题

**L_ans 的梯度无法穿过 KV cache 回传到 latent hiddens。**

这意味着：
- `L_trans` 是唯一训练 Student 潜空间表示质量的信号
- `L_ans` 只训练了 "给定 KV cache 后如何生成 answer"，但不约束 KV cache 本身的质量
- Student 的 latent hiddens 只被 Translator 的还原目标驱动，没有被 "最终答案正确性" 驱动

### 走捷径的表现

1. **前面 latent 过度优化还原能力**：Student 把所有信息都编码成"方便 Translator 还原文字"的形式，但这不一定是"方便后续生成正确答案"的形式
2. **后续 answer 部分依赖的 KV cache 质量差**：因为 KV cache 的形成过程（Phase 2 latent forward）没有被 L_ans 约束
3. **`</think>` 结束时机错误时问题更严重**：如果模型在错误的步数退出潜空间，KV cache 中的信息可能完全不足以支撑正确 answer 生成

### 可能的改进方向

1. **让 L_ans 的梯度能回传到 latent hiddens**：
   - 方案 A：Phase 3 不用 KV cache，而是把最后一个 latent hidden 拼接到 answer_embeds 前面，这样梯度可以直接回传
   - 方案 B：用 "straight-through" 技巧让梯度穿过 KV cache（实现复杂）

2. **加强 `</think>` 退出信号**：
   - 对 h_K（最后一个 latent）加正向 loss：鼓励 lm_head(h_K) 输出 `</think>`
   - 当前 aux loss 只对中间 latent 施加负向约束（不要输出 `</think>`），但对最后一步没有正向约束

3. **数据层面**：
   - 排查 GT answer 与视频的对应关系
   - 确认感知教师生成的 `<result>` 内容是否基于正确的视频

---

## 总结：当前 loss 设计的主要漏洞

| 漏洞 | 影响 | 严重程度 |
|------|------|---------|
| L_ans 梯度无法回传到 latent hiddens | Student 潜空间只被 Translator 驱动，不被答案正确性驱动 | ⚠️ 高 |
| L_ans 混合了 `</think>` 和 answer 内容 | 无法分别观察和调优两个目标 | 中 |
| L_aux 对 K=1 样本无效 | 大部分样本的"何时停止"信号缺失 | ⚠️ 高 |
| Translator 太强可能偷懒 | Student hidden 不需要真正编码信息 | 中 |
| 数据可能有 bug | 即使架构正确也学不到正确答案 | ⚠️ 高 |

## 已实施的修改（v5.1）

### 修改 1：冻结 Translator（方案 B）

**文件**：`training/stage1_sft_v5.py`

- Translator 加载后设为 `eval()` + `requires_grad_(False)`
- 从 DeepSpeed/优化器中移除 Translator 参数
- 不再保存 Translator checkpoint

**效果**：
- 防止 Student 和 Translator 共谋偷懒
- Translator 作为"固定评判者"，Student 必须把 h_i 编码成"预训练 LLM 能理解的格式"
- 省显存（Translator 不需要梯度和优化器状态）
- 梯度仍穿过 h_i 回传到 Student（因为 h_i 是 Student 的输出）

### 修改 2：新增 L_think_end loss

**文件**：`training/losses.py`

- `compute_think_end_loss(latent_hiddens, lm_head)`：对 h_K 施加正向 CE loss，目标为 `</think>` token
- `compute_aux_loss` 只对 h_1~h_{K-1} 压低 `</think>` 概率
- `compute_total_loss` 新增 `l_think_end` 和 `lambda_think_end` 参数

**效果**：
- 直接约束"最后一个 latent 应该输出 `</think>`"
- 与 `L_aux`（中间 latent 不应输出 `</think>`）形成互补
- wandb 上新增 `train/loss_think_end` 指标

### 修改 3：Phase 2/3 临时关闭 gradient checkpointing（关键修复）

**文件**：`training/latent_forward.py`

- `phase2_serial_forward`：进入前关闭 gradient checkpointing，退出后恢复
- `phase3_forward`：同上

**原因**：
- gradient checkpointing 开启时，transformers 会把 `use_cache=True` 覆盖为 `False`
- 这导致 KV cache 中的 tensor 不保留完整计算图
- L_ans 的梯度无法通过 attention → KV cache → latent hiddens 回传
- Phase 2 每步只 forward 1 个 token，Phase 3 answer 通常只有几十个 token，关闭 gradient checkpointing 不会导致 OOM

**效果**：
- L_ans 的梯度能有效回传到 Phase 2 的所有 latent hiddens
- 潜空间推理内容被"答案正确性"直接约束
- Phase 1（几千个 token）仍保留 gradient checkpointing 节省显存

### 当前 Loss 架构（v5.1）

```
L_total = λ_trans * L_trans + λ_ans * L_ans + λ_aux * L_aux + λ_think_end * L_think_end

梯度流向：
  L_trans     → Translator forward（冻结）→ 梯度穿过 h_i → Student ✅
  L_ans       → Phase 3 attention → KV cache → Phase 2 latent hiddens → Student ✅（修复后）
  L_aux       → lm_head(h_1~h_{K-1}) → Student ✅
  L_think_end → lm_head(h_K) → Student ✅

默认权重：λ_trans=1.0, λ_ans=1.0, λ_aux=0.1, λ_think_end=1.0
```
